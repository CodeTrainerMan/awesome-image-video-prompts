### 🤝 如何贡献

我们欢迎各种形式的贡献！无论您是添加新的提示词、改进文档、修复错误还是提出新功能，您的帮助都会让这个项目变得更好。

### 📋 贡献方式

1. **添加新提示词**：分享您创建或发现的高质量提示词
2. **改进组织方式**：建议更好的分类或文件夹结构
3. **修复问题**：报告错误或修复现有问题
4. **完善文档**：改进 README、添加示例或澄清说明
5. **审查拉取请求**：帮助审查和测试他人的贡献
6. **建议功能**：提出新想法或改进建议

### 🚀 开始贡献

#### 1. Fork 仓库

在 GitHub 仓库页面点击 "Fork" 按钮，创建您自己的副本。

#### 2. 克隆您的 Fork

```bash
git clone https://github.com/YOUR_USERNAME/awesome-image-video-prompts.git
cd awesome-image-video-prompts
```

#### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-fix-name
```

#### 4. 进行更改

- 按照格式标准添加新提示词
- 如需要，更新文档
- 如果可能，测试您的提示词

#### 5. 提交更改

```bash
git add .
git commit -m "Add: 您的更改描述"
```

**提交信息指南：**
- 使用清晰、描述性的消息
- 使用类型前缀：`Add:`、`Fix:`、`Update:`、`Docs:`、`Refactor:`
- 示例：
  - `Add: 风景摄影提示词`
  - `Fix: README 中的拼写错误`
  - `Update: 提示词格式规范`
  - `Docs: 改进贡献指南`

#### 6. 推送到您的 Fork

```bash
git push origin feature/your-feature-name
```

#### 7. 创建拉取请求

1. 前往 GitHub 上的原始仓库
2. 点击 "New Pull Request"
3. 选择您的 fork 和分支
4. 填写拉取请求模板
5. 提交 PR

### 📝 提示词贡献指南

#### 文件命名规范

- 格式：`分类-风格-标签.txt` 或 `分类-风格-标签.md`
- 使用小写字母和连字符
- 示例：
  - `portrait-photorealistic-4k.txt`
  - `landscape-cinematic-sunset.md`
  - `character-anime-fantasy.txt`

#### 提示词文件结构

每个提示词文件应遵循以下结构：

```markdown
# 提示词标题

## 描述
简要描述此提示词生成的内容。

## 提示词
[您的提示词文本]

## 负面提示词（可选）
[生成时要避免的内容]

## 参数（可选）
- 宽高比：16:9
- 模型：Stable Diffusion v1.5
- 步数：50
- CFG 比例：7

## 使用说明（可选）
使用此提示词的其他说明或技巧。

## 标签
摄影, 风景, 日落, 4k
```

#### 质量标准

- ✅ **清晰具体**：提示词应该定义明确
- ✅ **已测试**：如果可能，提交前测试提示词
- ✅ **已分类**：将提示词放在适当的文件夹中
- ✅ **已文档化**：包含描述和使用说明
- ✅ **原创或已署名**：如果使用他人的提示词，请提供署名

#### 分类

按以下结构组织提示词：

```
prompts/
├── image/
│   ├── artistic/          # 艺术风格
│   ├── photography/       # 摄影
│   ├── digital-art/       # 数字艺术
│   ├── product/           # 产品
│   ├── architecture/      # 建筑
│   └── fashion/           # 时尚
└── video/
    ├── cinematic/         # 电影风格
    ├── animation/         # 动画
    ├── commercial/        # 商业
    ├── social-media/      # 社交媒体
    └── educational/       # 教育
```

### 🔍 行为准则

#### 我们的标准

- 尊重和包容
- 欢迎新人并帮助他们学习
- 专注于建设性反馈
- 尊重不同的观点和经验

#### 不可接受的行为

- 骚扰或歧视性语言
- 恶意评论或侮辱性评论
- 发布他人的私人信息
- 其他可能被认为不当的行为

### 📋 拉取请求流程

1. **更新文档**：如果您添加功能，请更新 README
2. **遵循格式标准**：确保您的提示词遵循命名和结构约定
3. **测试您的更改**：如果可能，提交前测试提示词
4. **编写清晰的描述**：说明您的 PR 做了什么以及为什么
5. **保持耐心**：维护者会尽快审查您的 PR

#### PR 模板

创建拉取请求时，请包括：

- **类型**：什么类型的贡献（提示词、文档、修复等）
- **描述**：此 PR 做了什么？
- **测试**：如何测试的？
- **检查清单**：
  - [ ] 遵循文件命名约定
  - [ ] 包含描述和使用说明
  - [ ] 放在适当的分类中
  - [ ] 如需要，已更新文档

### 🐛 报告问题

#### 报告前

1. 检查问题是否已存在
2. 搜索已关闭的问题以查找类似问题
3. 确保您使用的是最新版本

#### 如何报告

1. 使用问题模板
2. 提供清晰的标题和描述
3. 包括重现步骤（如果适用）
4. 如有帮助，添加截图或示例
5. 指定您的环境（操作系统、工具等）

### 💡 建议和功能请求

我们欢迎建议！提出新功能时：

1. 检查是否已被建议过
2. 解释用例和好处
3. 考虑实现复杂性
4. 对讨论和反馈持开放态度

### 🎯 优先领域

我们特别欢迎以下方面的贡献：

- 高质量、经过测试的提示词
- 针对不太常见的 AI 工具的提示词
- 文档改进
- 翻译和本地化
- 错误修复和优化

### ❓ 有问题？

- 提出问题请打开 issue
- 查看现有问题和讨论
- 耐心等待回复

### 🙏 致谢

贡献者将：
- 在 README 中列出（如果愿意）
- 在发布说明中署名
- 受到社区的赞赏！

感谢您的贡献！🎉
